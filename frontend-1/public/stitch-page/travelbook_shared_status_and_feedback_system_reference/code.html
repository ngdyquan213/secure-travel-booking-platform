1. Shared Status and Feedback Family

All components utilize the Manrope display font for titles and Inter for supporting text. Surfaces are built using surface-container and surface-container-lowest to ensure tonal layering without harsh borders.

2. Semantic Mapping Rules

  • Success: Teal (#006A6A) - For affirmative outcomes only.
  • Warning: Amber (#E56C3E) - For time-sensitive or advisory info.
  • Error: Red (#BA1A1A) - For blocking issues and data failures.

3. Urgency and State Transitions

Transient feedback (Toasts) should persist for 3.5 seconds unless a 'Retry' action is present. Permanent states (Empty/Error cards) must provide at least one primary way out (CTA) to avoid user dead-ends.

4. Shared Form Continuity

Inline validation must be mapped 1:1 with the Status Badge family. Use `on-error` backgrounds for input highlighting and matching icons to ensure accessibility compliance (WCAG 2.1 AA).
