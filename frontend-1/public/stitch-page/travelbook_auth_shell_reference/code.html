Authentication Shell Reference Guide

Reference Rule 01

Background Treatment

The background utilizes a subtle tonal shift instead of flat color or noise. This "Azure Horizon" effect creates a sense of depth and focus, guiding the eye toward the central interaction point. No 1px lines are permitted for page-level sectioning.

  • check_circle Use surface-bright to surface-container radial gradient.
  • check_circle Prohibit textures or photographic backgrounds behind the auth card.

Reference Rule 04

Form & Validation States

Inputs use surface-container-highest backgrounds to eliminate the heavy "boxed" look. Errors are presented with a 20% opacity red container and semantic iconography.

Reference Rule 05

Support Microcopy Zone

The footer of the auth card is reserved for secondary navigation and microcopy. Use label-md with 0.05rem letter-spacing to maintain the editorial journal feel.

Reference Rule 06

Mobile Response Behavior

On small screens (md:hidden), the card shadow and radius are suppressed to allow the card to take full-width viewport real estate. Branding remains persistent but scaled down.
